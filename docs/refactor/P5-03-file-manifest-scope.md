# P5-03. Область применения file-manifest

## Цель
Сузить использование `file-manifest.json`. Не отдавать его целиком клиенту без нужды.

## Контекст / проблема
`server/assets/file-manifest.json` (≈180KB) содержит полный список всех файлов с типами. Эндпоинт `server/api/filesystem/files.get.ts` отдаёт его **целиком**. Для навигации в Explorer используется `list.ts` (отдаёт только дочерние entity). Два параллельных пути, пересечение ответственностей.

## Затронутые файлы
- `server/api/filesystem/files.get.ts`
- `scripts/generate-file-manifest.ts`
- Клиенты, если есть, которые дергают `/api/filesystem/files`.

## Шаги
1. `grep` по `app/` — есть ли вызовы `/api/filesystem/files`.
2. Если нет — удалить эндпоинт `files.get.ts` вообще. Генерация `file-manifest.json` остаётся для будущей фичи поиска (см. backlog P7-02).
3. Если есть — оставить, но добавить предупреждение в комментарий и ограничить выдачу (например, отдавать только типы файлов, без `size` и прочих деталей).
4. В `AGENTS.md` (P7-01) отметить, что `file-manifest.json` используется **только** для будущего полнотекстового поиска.

## Критерии готовности
- [x] Эндпоинт `files.get.ts` либо удалён, либо ограничен.
- [x] Нет клиента, который ненужно тянет ~180KB.
- [x] Playwright smoke зелёные.
- [x] Размер бандла не изменился (если эндпоинт удалён).

## Проверка
- `grep -r "filesystem/files" app/ server/` — видим только упоминания в `AGENTS.md` и backlog.
